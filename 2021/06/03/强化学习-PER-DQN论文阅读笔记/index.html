Prioritized Experience Replay

论文地址

1. BACKGROUND

TD errors 提供了测量这些优先级(抽取样本的优先级)的一种方法(Van Seijen＆Sutton, 2013)(TD-errors 也被用作确定在何处选择，进行资源的优先级化机制)。 作者的方法使用了一种的类似的优先级方法，但是对于 model-free RL而不是 model-based。 此外，作者使用随机优先级，在学习来自样本的函数近似器时更加强大。

​ (Narasimhan et al., 2015) re-sampling 区别出两个 buckets, 一个是积极的，一个是消极的，并为每一个选择一个固定的比例去应用。

2. PRIORITIZED REPLAY

Using a replay memory leads to design choices at two levels: which experiences to store, and which experiences to replay (and how to do so).

2.1 A MOTIVATING EXAMPLE

作者举例，一个名叫"Blind Cliffwalk’"的环境，一个简单化的 rare rewards 探索挑战。只有 n states， 其 Env 需要一个 exponential(指数式) 规模数字的随机步数直到第一个reward出现；简而言之，在这个挑战中一个随机的动作序列将导致2−n2^{-n}2−n 的 reward. 更进一步说，最相关的转化(珍贵成功)隐藏在一种高度冗余失败的情况下。

作者使用此示例来突出两个 Agent 的学习时间之间的差异。 两个Agent 对从同一 replay memory 中抽取 transitions 进行 Q-Learning 更新。 第一 Agent 以 random 地 replay memory，而第二 Agent 调用 oracle 以优先考虑 transitions。This oracle greedily selects the transition that maximally reduces the global loss in its current state.

2.2 PRIORITIZING WITH TD-ERROR

优先级重放的核心部分是衡量每个转换重要性的标准。一个理想化的标准是RL代理在其当前状态（预期学习进度）下可以从转换中学习的量。虽然这个量无法直接获得，但有一个合理的替代标准，一个转化的TD误差δ\deltaδ， 其可以表明这个 value 里下一个相互联系的估计值有多远。这特别适用于增量、online RL算法，如SARSA或Q-learning，这些算法已经计算TD误差并按比例更新参数δ\deltaδ。在某些情况下，TD误差也可能是一个很差的估计，例如，当奖励是嘈杂的。

为了证明通过 TD error 对重播进行优先级排序的潜在有效性，作者将 “Blind Cliffwalk’” 中的uniform和oracle基线与 greedy TD-error prioritization 算法进行了比较。此算法将最后遇到的 TD error 以及每个转换存储在 replay memory 中。从存储器中重放具有最大绝对 TD error 的转换。Q-learning 更新被应用于这个转换，它按照 TD error 的比例更新权重。新的转换到达时没有已知的 TD error，因此将它们放在最高优先级，以保证所有体验至少被看到一次。图2（左）显示，该算法大大减少了解决 “Blind Cliffwalk’” 任务所需的工作。

2.3 STOCHASTIC PRIORITIZATION

然而，greedy TD-error prioritization 有几个问题。首先，为了避免对整个 replay memory 进行昂贵的扫描，TD error 只针对重放的转换进行更新。一个结果是，在第一次访问时具有低 TD error 的转换可能很长时间内不会被重放（这意味着使用滑动窗口重放内存时实际上永远不会重放）。其次对噪声敏感。

为了克服这些问题，作者引入了一种在纯贪婪优先排序和均匀随机抽样之间插值的随机抽样方法。确保被采样的概率在转换的优先级中是单调的，同时保证即使对于最低优先级的转换也是非零的概率。具体地说，将 transition ammplingiii 的概率定义为:

3. 要点

这种方法的重点在于用一种优先级来抽取 replay memory 中的样本，替代简单的随机抽样。

样本的优先级是用 TD error 来确定，即 Q现实 - Q估计。很直观的可以体现预测值与真实值之间的差距，一句需要去学习的重要程度。每次抽取的样本是根据 TD error 确定的样本优先级 p，为了减少排序产生的消耗，即SumTree。SumTree是一个二叉树形结构，中间节点存储每个样本的优先级 p，叶子结点存储样本的 p， 跟节点为所有 p 的和。抽样过程就如同二叉树查找值一样。
